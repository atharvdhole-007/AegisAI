# AegisAI Hackathon Demo Script (5 Minutes)

**Step 1: Open Dashboard** 
→ *Judges see a dark-mode command center with live logs streaming linearly from the Apache Kafka backend, real-time XGBoost anomaly charts rendering in Recharts, and the Threat Stats updating immediately.*

**Step 2: Inject "APT Lateral Movement" Attack** 
→ *Judges see the log feed spike with high-severity red alerts. The Recharts anomaly confidence score spikes near 95%, and an AI Threat Detection Card appears detailing the attack narrative and affected internal subnets.*

**Step 3: Click "Generate Playbook" on the Threat Card** 
→ *Judges see the application seamlessly transition to the Interactive Playbook Canvas. A custom ReactFlow graph is rendered mapping the exact incident response steps (Investigate, Isolate, Eradicate) generated dynamically via Llama-Primus.*

**Step 4: Mark 2-3 Playbook Nodes as Complete** 
→ *Judges see the graph nodes dynamically change color from yellow (Pending) to green (Completed), proving the playbook is not just static text but an interactive command flowchart.*

**Step 5: Open Copilot Sidebar & Ask "Why did AI recommend isolation?"** 
→ *Judges see the sidebar slide in. As you type, the Grok API (or Llama) responds via SSE (Server-Sent Events) streaming, reading the exact topology context and explaining the lateral movement risks across the DMZ firewall.*

**Step 6: Navigate to Simulation Page**
→ *Judges see the dynamic NetworkX topology map of a simulated Union Bank architecture, mapping external internet down to core banking databases.*

**Step 7: Select "APT Full Kill Chain" → Launch Attack**
→ *Judges see the Red vs Blue team panel updating in absolute real-time. The map nodes flash as the Red Team (powered by RedSage) attempts exploits, and the Blue Team counters alongside logs highlighting MITRE TTPs.*

**Step 8: Simulation Concludes**
→ *Judges see the final Breach and Attack Simulation (BAS) report generated highlighting exactly what vulnerabilities allowed the exfiltration and the exact security configurations to patch.*
